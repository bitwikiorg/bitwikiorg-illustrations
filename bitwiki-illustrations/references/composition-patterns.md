# Composition Patterns

## 1. System Triad

Use for BIThub × BITwiki × BITCORE overviews.

- Three primary structures in a stable triangle.
- BIThub as active forum chamber.
- BITwiki as codex, library, or semantic lattice.
- BITCORE as octagram, hypercube, or orthogonal reasoning engine.
- Spectral flow connects all three; monochrome structure remains dominant.

## 2. Lifecycle Conveyor

Use for the canonical intelligence lifecycle.

- Left-to-right or circular sequence.
- Six discrete stages.
- Each stage has one object and one action.
- Return loop shows compounding reuse.

## 3. Workcell Cross-Section

Use for BIThub topics becoming operational workcells.

- Cutaway cube or octagonal chamber.
- Humans, agents, tools, context, and outputs visibly occupy different stations.
- Input enters through one bounded portal; artifacts leave through another.

## 4. Semantic Lattice

Use for BITwiki memory, ontology, provenance, or linked knowledge.

- Nodes are structured knowledge units, not generic glowing dots.
- Edge classes should differ visibly: semantic relation, provenance, workflow dependency, or temporal lineage.
- A codex or canonical core may anchor the lattice.

## 5. Orthogonal Engine

Use for BITCORE.

- A cubic human-readable frame contains or confronts a higher-order form.
- The BITCORE form breaks ordinary 3D alignment without becoming visually chaotic.
- Show recursive layers, tool routes, agents, and memory interfaces.

## 6. Cathode–Anode Transfer

Use for human–AI symbiosis.

- Human side: grounded, angular, bounded, tactile.
- AI side: radiating, orthogonal, multi-dimensional.
- A visible arc or spark marks knowledge transfer.
- Neither side dominates; the completed circuit is the subject.

## 7. Registry Atlas

Use for system inventory and discoverability.

- Central map or instrument panel.
- Peripheral classes: agents, models, tools, COREs, constructs, workspaces, artifacts, guides.
- Emphasize reference relationships rather than execution flow.

## 8. Artifact Refinery

Use for turning raw conversation into stable knowledge.

- Raw inputs enter as noisy fragments.
- Processing stages add scope, provenance, structure, validation, and versioning.
- Final outputs become durable modular objects or codex entries.

## 9. State Plate

Use for current authoritative condition.

- One bounded entity at one explicit time.
- Separate active, allowed, blocked, objective, and next-action zones.
- Avoid historical narrative unless shown as a secondary lineage.

## 10. Guardian Seal Plate

Use sparingly for covers, dividers, or identity anchors.

- One abstract ophanimic seal.
- Nested orbital wheels and eye-like flares.
- Spectral plasma concentrated near the emblem.
- No prose explanation inside the image.
