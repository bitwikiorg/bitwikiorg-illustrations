# Prompt Template

Use this sequence when constructing an image-generation prompt.

## 1. Semantic objective

State exactly what the illustration must explain.

```text
Explain [single operational concept] in the BITwiki ecosystem.
The image must show [required entities] and the relation [required transition or structure].
```

## 2. Canonical terminology

List only the necessary system terms and preserve their meaning from `ecosystem-ontology.md`.

## 3. Composition

Choose one pattern from `composition-patterns.md` and define:

- orientation
- primary focal object
- input and output directions
- bounded areas
- feedback or reuse loop
- reserved negative space, if labels will be added later

## 4. Geometry mapping

Specify the semantic geometry:

```text
Use cubes for bounded workcells, an octagram or hypercube for BITCORE,
a linked lattice for BITwiki, and a spectral arc only for active cognition transfer.
```

## 5. House style

### Modern technical monochrome clause

```text
Modern BITwiki visual language: ivory, white, graphite, charcoal, layered grey,
precise architectural linework, clean engineering grid, restrained modern typography,
minimal spectral ultraviolet accents for active intelligence only.
```

### Intelligence Codex clause

```text
Render as a medieval-to-17th-century engineering and scientific codex plate:
aged ivory engineering paper, dense technical woodcut linework, measured construction marks,
precise instruments and marginal diagrams, contemporary semantic content, no gothic typography,
full-spectrum ultraviolet energy restricted to active cognitive flows and state transitions.
```

## 6. Text policy

```text
Do not render paragraphs. Use no text unless required.
When labels are required, use only these exact short labels: [labels].
Leave clean annotation space so final typography can be added outside the image model.
```

## 7. Identity and exclusion clause

```text
Use only BITwiki identity and terminology. Do not include external author names,
handles, contact information, signatures, QR codes, foreign mascots, or copied compositions.
Do not add watermarks or presentation software branding.
```

## 8. Output clause

```text
16:9 landscape editorial illustration, high resolution, balanced for article placement,
semantically legible at reduced size, no slide frame, no mock presentation chrome.
```

## Example

```text
Explain how a BIThub topic becomes a reusable BITwiki artifact.
Show six stages: task enters BIThub; topic becomes a bounded workcell;
Agents, Constructs, and COREs process the task; useful output becomes an Artifact;
stable knowledge moves into BITwiki; future humans and agents reuse it.

Use a Lifecycle Conveyor composition. Represent the workcell as an open cube,
BITCORE processing as an octagram inside an octagonal engine chamber,
and BITwiki as a linked codex lattice. Use a closed return loop to show compounding reuse.

Render in the Intelligence Codex house style: aged ivory engineering paper,
dense technical woodcut linework, exact construction marks, contemporary system semantics,
and concentrated ultraviolet full-spectrum energy only along the active transfer path.

No paragraphs inside the image. Reserve six clean label zones.
Use only BITwiki identity. No watermarks, external branding, signatures, or copied mascots.
16:9 landscape editorial illustration.
```
